# Домашнее задание к лекции «Работа с SQL. Создание БД»
## Часть 1: Разработка схемы базы данных для музыкального сервиса
### Сущности и связи:
#### Artists (Исполнители)

- Каждый исполнитель имеет уникальный ID и имя

- Один исполнитель может принадлежать к нескольким жанрам

- Один исполнитель может участвовать в нескольких альбомах

#### Genres (Жанры)

- Каждый жанр имеет уникальный ID и название

- Один жанр может включать несколько исполнителей

- Связь с Artists: многие-ко-многим

#### Albums (Альбомы)

- Каждый альбом имеет уникальный ID, название и год выпуска

- Один альбом может содержать треки нескольких исполнителей

- Связь с Artists: многие-ко-многим

#### Tracks (Треки)

- Каждый трек имеет уникальный ID, название, продолжительность

- Трек принадлежит строго одному альбому

- Один трек может входить в несколько сборников

#### Collections (Сборники)

- Каждый сборник имеет уникальный ID, название и год выпуска

- В сборник могут входить треки из разных альбомов

- Связь с Tracks: многие-ко-многим

## Особенности реализации

1. Целостность данных: Использованы внешние ключи с каскадным удалением
2. Нормализация: Таблицы нормализованы до 3NF
3. Производительность: Primary keys и Foreign keys для быстрого поиска
4. Гибкость: Поддержка многих-ко-многим отношений
5. Расширяемость: Легко добавить новые сущности (например, плейлисты)
